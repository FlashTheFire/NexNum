# Audit Secrets Report (NexNum) — Redacted

## Scope
Backend and repo-wide check for committed secrets and sensitive configuration.

## Findings

- .env and .env.* are listed in .gitignore; no .env files are tracked.
- JWT_SECRET and ENCRYPTION_KEY are read from process.env only; no fallback secrets in repo (dev fallback is gated by ALLOW_DEV_JWT).
- No hardcoded API keys, private keys, or passwords found in src/ or config files scanned.
- Payment/UPI and provider auth keys are loaded from environment (payment-settings, provider config from DB with encrypted auth key).

## Remediation

- Keep all secrets in environment or a secrets manager; do not commit .env.
- In CI, provide JWT_SECRET (and optionally ALLOW_DEV_JWT=true for test jobs).
- Rotate any keys if they were ever committed in the past (check git history if needed).

## Redacted examples (patterns only)

- process.env.JWT_SECRET — OK, env-only.
- process.env.ENCRYPTION_KEY — OK, env-only.
- Bearer ${process.env.CRON_SECRET} — OK, env-only.

No actual secret values appear in this report.
